iv ABSTRACT  
The rapid proli feration of AI -generated media  including deepfake images produced by Stable 
Diffusion and DALL -E, cloned voices synthesised by XTTS and VITS, machine -generated phishing text 
authored by GPT -4, and face -swapped v ideos  poses serious and escalating  challenges to digital trust, 
cybercrime investigation, and public safety. Existing detection tools require manual content submission 
and are limited to single modalities, leaving citizens unprotected during everyday application usage. 
RiskGuard addresses these challenges through a comprehensive, production -grade, multi -modal detection 
platform that identifies synthetic content across four modalities and preserves forensic evidence on a 
public blockchain.The system employs a hybrid CPU + GPU architecture co mbining lightweight, zero -
dependency local signal processing with cloud -based transformer models. The image detection engine 
uses a 6 -signal type -adaptive weighted ensemble incorporating Noise Print Residual analysis, DCT 
spectral fingerprinting, Haar wave let decomposition, perceptual hashing, and cloud -based CNN and 
HuggingFace classifiers. A structural classifier dynamically adjusts signal weights based on image type 
(photograph, digital art, or screenshot), preventing false positives on non -photographic content. The voice 
detection pipeline uses a 6 -signal hybrid architecture with LFCC, CQT phase coherence, modulation 
spectrum analysis, pitch contour tracking, statistical moment features, and a wav2vec2 -base model fine -
tuned on ASVspoof2019 data. Text det ection combines DeBERTa and RoBERTa cloud classifiers with 
local statistical analysis and a Binoculars perplexity pro xy for zero -shot LLM detection. To ensure 
forensic integrity, detected evidence is secured using SHA -256 hashing, IPFS decentralised storage  via 
Pinata, and Merkle -tree batch anchoring on the Polygon Amoy blockchain through a custom 
EvidenceAnchor smart contract. Merkle batching reduces per -record blockchain costs by up to 99.7% 
compared to individual transactions. A Flutter mobile application  serves public users with real -time 
protection through a system overlay that monitors whitelisted applications (WhatsApp, Chrome, 
Instagram, Telegram, Facebook) for deepfake content, while a FastAPI -powered web dashboard provides 
law enforcement investigat ors with real -time, tamper -proof evidence review capabilities through Server -
Sent Events. Experimental results demonstrate 70 –92% accuracy on image detection, 65 –85% on voice 
detection, and 70 –88% on text detection, with real -time processing under 200ms pe r voice chunk and 
under 150ms for local image analysis.

1.1 OVERVIEW  
The proliferation of AI -generated media —such as deepfake images created using Stable 
Diffusion, cloned voices via XTTS/VITS, and AI -authored  phishing text powered by GPT -4      
poses an escalating threat to digital trust, cybercrime investigation, and public safety. The rapid 
advancement of generative AI technologies has made it increasingly difficult for both humans and 
traditional software systems to distinguish authentic content from synthetic media. This has crea ted 
unprecedented challenges for law enforcement agencies, financial institutions, and ordinary citizens 
who depend on the reliability and authenticity of digital communications.  
RiskGuard addresses this growing threat landscape by offering a comprehensive  multi -
modal detection platform that integrates local signal processing with cloud -based AI models. The 
system is designed as a dual -interface solution, featuring a Flutter -based mobile application for 
citizen -facing threat detection with real -time protect ion overlay, alongside a FastAPI -powered web 
dashboard for law enforcement evidence management. This hybrid architecture ensures 
accessibility and performance even on commodity hardware, while also maintaining forensic -grade 
evidence integrity through bloc kchain anchoring.  
1.2 PROBLEM STATEMENT  
            Law enforcement agencies face four core challenges in the current digital landscape. The 
first is detection, which involves accurately distinguishing AI -generated content from authentic 
media across multi ple modalities such as images, voice, text, and video, while maintaining low 
latency. The second challenge is evidence preservation, where agencies must ensure a tamper -proof 
chain of custody for digital forensic evidence that can withstand legal scrutiny in courts. These 
challenges are further complicated by the increasing sophistication of generative AI technologies, 
which make synthetic content more realistic and harder to identify.  
          In addition to detection and preservation, accessibility and r eal-time protection remain  
critical concerns. There is a growing need to provide real -time detection tools that are easily 
accessible to both citizens and investigators, enabling proactive identification of threats without 
requiring manual initiation. How ever, most existing solutions rely on users to manually submit

            RiskGuard addresses the key challenges in detecting AI -generated threats through a robust 
multi -signal ensemble architecture applied across different data modalities. This appro ach achieves

LITERATURE SURVEY  
2.1 Noiseprint : A CNN -Based Camera Model Fingerprint                     
Cozzolino & Verdoliva (2020)  
       This foundational work in camera model fingerprinting introduces an advanced method for 
identifying the source of digital images by extracting intrinsic sensor n oise patterns. The authors 
propose the concept of a Noise Print Residual (NPR), which is obtained by computing the difference 
between an original image and its median -filtered version. This process effectively isolates camera -
specific noise artifacts that are typically invisible to the human eye but remain consistent across 
images captured by the same device. These unique noise patterns serve as reliable fingerprints for 
distinguishing between images from different cameras.  
       AI-generated images, parti cularly those produced by modern generative models, lack these 
natural sensor noise characteristics, making NPR a powerful tool for detection. In the RiskGuard 
system, this technique is adapted as one of the key signals in the image detection ensemble. The  
system analyzes features such as noise variance, block consistency, and spectral flatness to 
differentiate authentic photographs from synthetic images. This approach significantly enhances 
the robustness of image -based deepfake detection.  
2.2 End-to-End A nti-Spoof ing with RawNet2                                                    
Tak et al. (2021)  
       This research presents a novel end -to-end framework for detecting spoofed audio using raw 
waveform inputs rather than traditional handcrafted features. Th e study highlights the effectiveness 
of Constant -Q Transform (CQT) in analyzing phase coherence, which reveals subtle artifacts 
introduced by neural vocoders in synthesized speech. Unlike conventional Short -Time Fourier 
Transform (STFT) representations, CQ T provides a more detailed frequency resolution, making it 
suitable for detecting inconsistencies in generated audio signals.  
       RiskGuard incorporates this concept by utilizing CQT -based phase coherence analysis within 
its voice detection pipeline. Sp ecifically, it measures phase derivative variance to identify

4 unnaturally consistent phase patterns that are characteristic of text -to-speech and voice cloning 
systems. This method allows the system to detect even highly realistic synthetic audio, thereby 
improving the accuracy and reliability of voice deepfake detection.  
 
2.3 wav2vec 2.0: Self -Supervised Learning of Speech Representations  
 Baevski et al. (2020)  
       This seminal work introduces wav2vec 2.0, a self -supervised learning framework designed to 
extract meaningful speech representations from large amounts of unlabeled audio data. The model 
leverages contrastive learning techniques to capture contextua l information in speech signals, 
making it highly effective for a wide range of speech processing tasks. Its ability to learn robust 
features without requiring extensive labeled datasets has made it a widely adopted standard in 
speech recognition and anti -spoofing applications.  
       In the RiskGuard system, a wav2vec2 -base model is fine -tuned using the ASVspoof2019 
dataset to enhance its capability in detecting synthetic speech. The model is deployed via ONNX 
export on a GPU -enabled environment, contribut ing a significant weighted signal to the voice 
detection framework. This cloud -based component complements multiple lightweight CPU -based 
signals, ensuring a balanced and efficient approach to deepfake voice detection.  
2.4 Spotting LLMs with Binoculars  
Hans et al. (2024)  
        This recent study proposes a zero -shot detection approach for identifying AI -generated text  
by comparing perplexity scores derived from two different language model perspectives. The 
method leverages differences in probability dis tributions to distinguish between human -written and 
machine -generated text without requiring task -specific training. This makes it a flexible and 
generalizable solution for detecting outputs from various large language models.  
        RiskGuard implements a simplified version of this concept using unigram and bigram entropy 
ratios as proxy measures for perplexity. This lightweight approach enables efficient detection of 
AI-generated text without the need for computationally intensive models. It acts as a un iversal

5 signal that complements more specialized classifiers such as DeBERTa and RoBERTa, thereby 
strengthening the overall text detection ensemble.  
2.5 Design, Analysis and Experimental Evalu ation of Block Level Features 
Sahidullah et al. (2015)  
       This work provides a comprehensive analysis of statistical features for detecting spoofed 
speech, emphasizing the importance of higher -order amplitude moments such as kurtosis and 
skewness. The study demonstrates that these statistical properties di ffer significantly between 
natural and synthetic audio signals due to variations in signal generation processes. These features 
serve as effective indicators for identifying anomalies in speech data.  
       RiskGuard incorporates these statistical measures  as part of its voice detection pipeline to 
enhance robustness. By analyzing amplitude distributions and identifying deviations from natural 
speech patterns, the system can detect synthetic audio even when advanced detection methods are 
unavailable. This l ightweight feature set acts as a reliable backup signal, ensuring consistent 
performance across different scenarios.  
2.6 ASVspoof 5: Crowdsourced Sp eech Synthesis Detection. ISCA Interspeech  
ASVspoof Consortium (2024).  
       The ASVspoof 5 challenge repre sents a significant advancement in the field of automatic

this edition introduces a crowdsourced approach to speech synthesis detection, incorporating a 
diverse ra nge of spoofing attacks generated using modern text -to-speech (TTS) and voice 
conversion techniques. The dataset includes realistic and varied audio samples collected from 
multiple sources, making it more representative of real -world scenarios. This divers ity enhances the 
robustness of evaluation frameworks and encourages the development of generalized anti -spoofing 
models capable of detecting unseen attacks.  
       In the RiskGuard system, insights from ASVspoof 5 are utilized to improve the effectiveness 
of voice deepfake detection. The system leverages similar evaluation strategies and incorporates 
training data inspired by such benchmark datasets to enhance model generalization. By aligning 
with the methodologies proposed in ASVspoof 5, RiskGuard ensures  that its detection pipeline

2.7 IPFS – Content Addressed, Ve rsioned, P2P File System. arXiv:1407.3561  
Benet, J. (2014).  
       This work introduces the InterPlanetary File System (IPFS), a decentralized peer -to-peer file 
storage system designed to make the web faster, safer, and more open. Unlike traditional HTTP -
based systems that rely on location -based addressing, IPFS uses content addressing, where each file 
is identified by a unique cryptographic hash. This ensures data integrity, as any modification to the 
file results in a completely different hash. Additionally,  IPFS supports versioning and distributed 
storage, allowing files to be stored across multiple nodes, thereby improving redundancy and 
availability while reducing dependency on centralized servers.  
       In the RiskGuard system, IPFS concepts are utilized  for secure and efficient storage of forensic 
evidence such as images, audio files, and analysis reports. By storing content using hash -based 
identifiers, the system ensures tamper -proof data management and easy verification of file 
authenticity. This appr oach also complements blockchain integration, where only the hash 
references are stored on -chain, significantly reducing storage costs while maintaining traceability 
and integrity. As a result, RiskGuard achieves a scalable and secure architecture for hand ling large 
volumes of digital evidence.

evidence systems operate independently of detection, creating a fragmented workflow where

provides real -time, proactive protection during everyday app usage such as browsing WhatsApp, 
Chrome, or Instagram.

applications and scans content as users enc ounter it during normal usage.

5.3 MODULE  DESCRIPTION  
5.3.1 IMAGE ANALYSIS MODULE  
The image detection engine uses a 6 -signal type -adaptive weighted ensemble architecture 
that classifies images into three types (photo, digital art, screenshot) and dynamically adjusts signal 
weights accordingly. A lightweight structural classifier (under 5ms, no ML model) determines 
image type based on colour count, gradient smoothness, EXIF metadata, and rectangular region 
detection.  
The NPR signal extracts camera sensor noise patterns by computing the residual between 
the original image and its median -filtered version. AI -generated images show unnaturally uniform 
noise, while real photos have sensor -specific fingerprints. The DCT spec tral analysis detects 
GAN/diffusion frequency -domain fingerprints by measuring spectral decay rate and uniformity. 
For digital art, DCT receives 65% of the total weight because cloud/ONNX models trained on 
photos produce false positives on artwork. Signals  returning None due to timeout or error are 
excluded and their weights redistributed proportionally among active signals, ensuring graceful 
degradation.

5.4.1 IMAGE ANALYSIS FLOW  
       When a user uploads or captures an image, the system initiates  the analysis by first classifying 
the input into categories such as photograph, digital artwork, or screenshot. This classification step 
is essential because different image types exhibit distinct characteristics, and applying a uniform 
detection strategy  may reduce accuracy. Based on the identified category, the system dynamically 
assigns appropriate weights to various detection signals. These signals are designed to capture 
different aspects of image authenticity, ensuring a more tailored and context -aware evaluation 
process.  
        All six detection signals are executed concurrently to optimize processing efficiency and 
reduce latency. In cases where certain signals fail or produce unreliable outputs, they are 
automatically excluded from the analysis, a nd the remaining weights are redistributed 
proportionally among the active signals. The final output is generated using a weighted ensemble 
approach, which produces an overall probability indicating whether the image is AI -generated. 
Additionally, the syst em provides a human -readable explanatio n to enhance interpretability.  
5.4.2 VOICE ANALYSIS FLOW  
The voice analysis pipeline begins with a comprehensive preprocessing stage to standardize 
the input audio for further analysis. This includes converting stereo audio into mono format, 
resampling the signal to 16 kHz for consistency, and applying Voice Activity Detection (VAD) to 
remove silent segments. These preprocessing steps are crucial for improving signal quality and 
ensuring that only relevant speech  segments are analyzed, thereby enhancing detection accuracy.  
For real -time applications, the system processes audio in small segments of approximately 
0.5 seconds, enabling low -latency analysis with each chunk processed in under 200 milliseconds. 
Multiple  detection signals operate in parallel, extracting features related to spectral, temporal, and 
phase characteristics of the audio. The results from these local signals are then combined with 
optional cloud -based inference using a wav2vec2 model deployed on  a GPU environment. This 
hybrid approach balances efficiency and accuracy, allowing the system to effectively detect 
synthetic or manipulated audio in real time.  

===== PAGE 33 =====
 
23 5.4.3 REAL -TIME OVERLAY FLOW  
The real -time overlay system is designed to provide seamless and continuous protection 
without interrupting the user’s workflow. When a user opens a whitelisted application, the system’s 
accessibility service detects the change in the foreground application and automatically activates an 
overlay bubble. This overlay ope rates as an intelligent assistant, monitoring user activity in real 
time while remaining unobtrusive. It continuously captures relevant contextual data such as URLs 
and triggers verification processes instantly.  
As the user interacts with media content, th e overlay captures the dominant image or video 
frame and performs deepfake analysis in the background. In the case of phone calls, the overlay 
adapts dynamically by transforming into a call monitoring interface that evaluates voice 
authenticity. The detect ion results are presented through an expandable card interface, allowing 
users to view detailed insights without leaving their current application. This integrated workflow 
ensures real -time threat detection while maintaining a smooth and user -friendly exp erience.  
5.4.4 EVIDENCE FILING FLOW  
       The evidence filing process is designed to ensure secure, tamper -proof storage and efficient 
verification of reported threats. When a user submits evidence, the system first generates a SHA -
256 cryptographic hash of the file, which serves as a unique digital fingerprint. The file is then 
uploaded to a decentralized storage system using IPFS via the Pinata service, ensuring redundancy 
and data integrity.  
       To optimize blockchain storage, the system aggregates multiple evidence hashes into a Merkle 
tree structure, where only the root hash is eventually recorded on -chain through a Polygon 
transaction. This batching approach significantly reduces transaction costs while maintaining full 
traceability of individual records.
